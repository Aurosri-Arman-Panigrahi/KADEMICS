🔑 Step 1: Get Your JWT Secret (C Backend Auth)

KADEMICS uses its own auth — no Google/GitHub OAuth needed. The C backend generates session tokens using a secret. In src/auth.c, add your secret:

// In src/auth.c
#define JWT_SECRET  "YOUR_SECRET_KEY_HERE_32_CHARS_MIN"
#define ADMIN_EMAIL  "2505022@kiit.ac.in"
#define ADMIN_PASS   "Kademics20180704"

Generate a secure secret: openssl rand -hex 32 in any terminal.

📧 Step 2: SMTP for Email Notifications (Optional)

To send email notifications when requests are approved/denied, configure SMTP credentials in the C backend:

// In src/config.c (not yet implemented — future)
#define SMTP_HOST    "smtp.gmail.com"
#define SMTP_PORT    "587"
#define SMTP_EMAIL   "your-kiit-email@kiit.ac.in"
#define SMTP_PASS    "your-app-password"

Get Gmail App Password: Google Account → Security → 2FA → App Passwords

☁️ Step 3: Deploy to Production (Optional)

To host KADEMICS online so all KIIT students can access it:

  1. Get a VPS: Railway.app (free tier) or Render.com
  2. Set environment variable: JWT_SECRET=your_secret_here
  3. Build C server: make → upload binary
  4. Point your domain (free: freenom.com) to the server IP

✅ Current Status — Demo Mode

KADEMICS is running in DEMO MODE right now. Everything works: login, archive drill-down, oracle ratings, quiz taking with anti-cheat, nexus chat — all via smart localStorage mock data. No external API key is needed for this mode.

When you run make run and the C server boots, the frontend auto-switches to real API mode.
